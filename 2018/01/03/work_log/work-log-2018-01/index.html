工作日志(2018-01)

发表于 2018-01-03 | 分类于 Work Log ， 2018 | 阅读次数

     年轻时: 哈哈哈, 新年喽, 又长了一岁; 老了后: 啊啊啊, 新年啊, 又老了一岁…

Wednesday 3rd      水曜日

     今天在使用 Android Studio 向模拟器中安装项目的时候, 总是失败, 提示.



     刚刚开始以为是原来的项目与现在安装的项目, debug-code 不同, 删除了重新安装, 还是不行. 然后就重新开了一个模拟器, 安装仍然不行, 后来发现其实是 .so 包的问题. 其实仔细看一下 ABIS 的问题, Application Binary Interface 的问题, 然后就可以定位到了 .so 包的原因. (坑啊)
     因为项目中的称重模块, 使用到了第三方的 .so 包, 基于 arm 架构. 而模拟器是基于 x86 架构的, 两者并不兼容, 所以会提示错误.
     解决办法. 1) 直接删除 .so 包. 2) 暂时还没有调试好, 可以指定相应的文件夹. 不过操作太高级, 玩不转.
